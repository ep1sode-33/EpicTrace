# Product

## Register

product

## Users

学生、知识工作者、工程师、PM、研究人员 —— 任何会**主动记录**自己工作 session 的人(上课、开会、线上讲座、项目调研、debug、看视频学习、做作业、写代码、混合多事的深度工作)。使用场景:在自己的电脑上(macOS 优先),**本地优先**;主动开 session 采集自己的多模态素材,事后再处理、整理、查询。界面为**中文**。单人本地使用,非团队/云端。

## Product Purpose

EpicTrace 是一个**本地优先的 AI session memory / 知识工作区**。它把用户主动记录的 session 和文件(音频→transcript、截图、剪贴板、笔记、文件)变成 **Project 级别的、可检索、可引用、可复用**的本地知识库。三个阶段:**采集 → 处理与入库 → 使用(项目与带引用对话)**。成功 = 用户能对自己采集的素材提问,并**跳回原始来源的那一刻**。

## Brand Personality

**冷静 · 专注 · 可信。** 一个安静、克制、会"让路"的工具——和"抢注意力"相反,让用户专心于自己的工作。语气:精确、不啰嗦、专业但不堆术语。情感目标:**安心 + 专注**(它替我记住了东西;我的数据始终是我的)。

## Anti-references

- **通用 AI/SaaS 套版**:紫/靛渐变、hero-metric 模板、千篇一律的"图标+标题"卡片网格、默认玻璃拟态、每个区块上的小写字母 eyebrow 标签、渐变文字、侧边色条边框。
- **cream/sand/米色暖中性底色**(2026 的 AI 默认套路)。
- 它要取代的**那个丑的两面板脚手架**(让用户往裸输入框里手打绝对路径)。
- 喧闹、繁忙、仪表盘信息过载的"指挥中心"风。
- **正面参照(取气质,不照抄)**:Zoom Workplace 顶栏(干净的图标+文字 tab,横跨三个产品阶段);Codex/Claude 桌面端对话面(安静侧栏 + 居中对话入口)。

## Design Principles

- **流程即导航**:顶栏三 tab(采集 / 信息处理和入库 / 项目与对话)本身就教会用户"采集→处理→使用"的流水线;结构传达用法。
- **安静服务于专注**:克制的表面、充裕留白、一个克制的强调色——界面绝不与用户的工作争夺注意力。
- **Agent 提议、用户掌控**:每个自动的归类/放置都可复核;不经用户确认绝不提交;UI 要让"提议中 vs 已确认"的状态一眼可辨。
- **本地优先、来源可溯**:界面映射真实的本地文件夹;回答/引用能跳回原始时刻——UI 暴露来源,而非黑盒。
- **诚实的状态**:待索引/已索引、busy/disabled、"开发中"占位都明确可信,绝不假装能用。

## Accessibility & Inclusion

个人项目,本期**不设特定 WCAG 目标**。只守合理默认:正文对比度可读(别用浅灰糊在浅底上)、**状态不只靠颜色**(徽章配文字)、表单可键盘操作。以后需要再加强。
