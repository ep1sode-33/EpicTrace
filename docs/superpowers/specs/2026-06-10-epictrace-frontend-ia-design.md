# EpicTrace 前端 IA 与界面设计

- **日期**:2026-06-10
- **范围**:前端**信息架构 + 入口逻辑 + 视觉方向**,以及为支撑它所需的**最小后端补充**。取代当前那个临时两面板脚手架(`ProjectPanel`/`IngestPanel`)。
- **背景**:首版前端是"调通后端用"的最小脚手架,未经设计——丑、且让用户手打绝对路径,入口错误。本 spec 把前端按产品三阶段重新设计。
- **参考**:Zoom Workplace 的顶栏 tab(图标+文字);Codex 桌面端的"项目侧栏 + 对话入口"。

---

## 1. 顶层 IA:顶栏三入口 = 产品三阶段

全局一个 **Zoom 式顶栏**,三个 tab(图标 + 文字),对应产品流水线的三个阶段(**采进来 → 处理入库 → 用起来**):

| Tab | 角色("三区心智模型") | 本期是否接通 |
|---|---|---|
| **采集** | 开 session 采集原始信息(麦克风/系统声音/截图/剪贴板/笔记/文件) | ⛔ 占位("开发中"),Plan 4 接通 |
| **信息处理和入库** | "收件箱/处理区":多媒体处理、切割、归类、**文件直接入库、快速创建项目、待索引列表** | ✅ 部分接通(见 §5) |
| **项目与对话** | "成品知识库":浏览各 Project 与其整理好的文件、**带引用对话**(Codex 式) | ✅ 项目/文件浏览接通;对话占位(Plan 3) |

**心智边界**:"往里进/正在处理"归 **信息处理和入库**;"已经在里面、拿来用"归 **项目与对话**(查看某项目入库了哪些文件,在这里看)。

## 2. Tab「采集」(本期占位)

本期渲染一个清晰的占位:简述"开 session 采集"将做什么 + 一个禁用的"开始 session"按钮 + "开发中"标识。完整实现见后续 Plan 4。不接任何采集后端。

---

## 3. Tab「信息处理和入库」(本期部分接通)

这是处理区。本期包含:

1. **快速创建项目**:一个 **`➕ 创建项目`**(加号 + 文字)按钮。点击 → **页面内弹出创建弹窗**(见 §6)。
2. **文件直接入库**:把外部文件登记进某个 Project(选目标 Project + 选文件 + 可选描述)。文件被**复制进 Project 文件夹**后落库(复用现有 `IngestService`)。文件选择走**原生文件选择器**(见 §7),不再手打路径。
3. **待索引列表**:列出各 Project 中**已入库但尚未建立 embedding 索引**的条目(包括:刚拖入复制的文件、以及创建项目时从非空文件夹扫描登记的现有文件)。每项显示来源/项目/状态=「待索引」。
   - 本期"处理/建索引"动作为**占位按钮**(实际 embedding/索引是 Plan 2)。
4. (占位)多媒体处理、切割、归类入口——后续 Plan 5–8 填入。

---

## 4. Tab「项目与对话」(Codex 式;本期项目/文件接通,对话占位)

布局 = **左侧项目侧栏 + 主工作区**(Codex 式):

- **左侧栏**:项目列表(点选切换);底部"➕ 新建项目"(同 §6 弹窗)。(未来:每个项目下挂会话列表,如 Codex 的 chats。)
- **主工作区(选中某项目时)**:
  - 顶部:项目标题 + 文件夹路径。
  - **文件/来源视图**:列出该项目已入库文件(文件名、大小、入库方式、是否已索引、描述)。
  - **对话区**:本期是**占位**——一个居中的"问点什么"输入框 + "对话功能开发中(需先建立索引)"提示。Plan 3 接通 Agentic RAG + 带引用回答。
- **空状态**:未选/无项目时,居中引导"创建第一个项目"。

## 5. 创建项目流程(核心入口,§3 与 §4 复用同一组件)

**入口**:信息处理和入库的 `➕ 创建项目`、项目与对话侧栏的 `➕ 新建项目`,两处复用同一个 **`CreateProjectModal`**。

**流程**:
1. 点 ➕ → 页面内弹出弹窗。
2. 弹窗内:**标题输入框** + **「选择文件夹」按钮**。
3. 点「选择文件夹」→ **跳出原生 Finder 文件夹选择器**(§7)→ 选中后回填路径显示在弹窗里(Finder 内可"新建文件夹")。
4. 确认创建。

**非空文件夹处理(已定)**:**接受非空文件夹**。
- 创建后,该文件夹里**已有的内容**经**扫描**后进入 **embedding 暂存区**,在「信息处理和入库」标为**「待索引」**(没建 embedding 就还不能被对话检索)。
- 这些"文件夹内既有文件"采用**就地索引**(不复制,入库记录直接指向其原路径);外部拖入/➕ 的文件则**先复制进**文件夹再登记。两种入库记录统一指向"文件在 Project 文件夹中的路径"。

**扫描的抓取范围(已定原则)**:绝不一股脑全收。
- 扫描时**默认套忽略规则**:尊重 `.gitignore`,并硬跳过 `node_modules`、`.git`、常见二进制、超大文件;默认只挑可索引的文本/文档类(md/txt/pdf/ppt/docx/代码等)。
- 把扫到的清单**列给用户复核**,允许勾掉不要的。
- (本期可先实现:扫描 + 套默认忽略 + 列出为待索引;"勾选增删"可作为同 Plan 内的增强。)

## 6. 原生文件/文件夹选择器(pywebview 桥)+ 开发态回退

- **打包态(pywebview 窗口内)**:通过 pywebview 的 **`js_api`** 暴露 Python 方法给前端:`pick_folder() -> str | None`、`pick_file() -> str | None`(内部调 `window.create_file_dialog(webview.FOLDER_DIALOG / OPEN_DIALOG)`)。前端通过 `window.pywebview.api.pick_folder()` 调用,拿回原生选中的绝对路径。`shell/run.py` 创建窗口时注入这个 `js_api`。
- **开发态(Vite 浏览器,无 pywebview)**:`window.pywebview` 不存在 → **回退**为一个普通文本输入(让开发者手填路径)。前端用 `window.pywebview?.api` 探测来决定用哪种。
- 设计上前端封装一个 `pickFolder()/pickFile()` helper 统一这两条路径,组件不感知。

## 8. 本设计需要的最小后端补充

1. **原生对话框桥**:`shell/run.py` 注入 `js_api`(`pick_folder`/`pick_file` 调 pywebview `create_file_dialog`)。
2. **文件夹扫描登记**:新增一个 service/endpoint,把指定 Project 文件夹中"既有文件"按忽略规则扫描,登记为**就地、待索引**的 `IngestRecord`(不复制)。
3. **`IngestRecord` 加状态**:新增 `indexed: bool`(默认 False)与一个区分入库方式的取值(现有 `ingest_method` 扩展:`file_direct`(复制) / `folder_scan`(就地))。已索引/待索引据此区分。
4. 现有 `IngestService`(复制+落库)与 projects/files 路由复用,新增上述扫描登记。

(以上 2–4 是本前端设计的连带后端改动,会进对应实现计划。)

## 9. 视觉方向

- **浅色主题**(贴合 Zoom/Codex 两个参考)。shadcn/ui + Tailwind v4。
- 顶栏:**图标 + 文字的 tab**,当前 tab 高亮(Zoom 式)。
- 项目与对话页:**Codex 式**——干净侧栏、居中的对话入口、克制的留白与排版层级。
- 强调:清晰的**空状态**、一致的间距/字号层级、按钮的 busy/disabled 态(沿用上轮加的错误处理与请求中禁用)。
- 目标:不再是"功能堆叠的丑脚手架",而是一个让人**一眼看懂三阶段、用着顺手**的桌面 app。

## 10. 组件结构(前端)

- `App.tsx` — 顶栏 + tab 路由(三 tab)。
- `components/TopBar.tsx` — Zoom 式三 tab。
- `views/CaptureView.tsx` — 采集占位。
- `views/ProcessIngestView.tsx` — 快速建项目入口 + 文件直接入库 + 待索引列表。
- `views/ProjectsConversationView.tsx` — 项目侧栏 + 项目工作区 + 对话占位。
- `components/CreateProjectModal.tsx` — 共享创建弹窗(标题 + 选择文件夹 + 非空提示)。
- `components/ProjectSidebar.tsx`、`components/FileList.tsx`、`components/PendingList.tsx`。
- `lib/api.ts` — 现有(相对 URL)+ 新增扫描/登记等端点。
- `lib/pickers.ts` — `pickFolder`/`pickFile`(pywebview 桥 + 开发态回退)。
- 删除/取代:旧的 `ProjectPanel.tsx`、`IngestPanel.tsx`。
